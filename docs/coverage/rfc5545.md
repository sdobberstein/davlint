- `not covered` — no test exists
- `deferred` — intentionally out of scope with reason

> **Scope note:** RFC 5545 defines the iCalendar format. Like RFC 2426/6350 for vCards, the majority of its requirements govern how a *client* generates iCalendar data. From a CalDAV server perspective, the testable surface is: **does the server accept valid iCalendar and reject invalid iCalendar on PUT**, returning `CALDAV:valid-calendar-data` or `CALDAV:valid-calendar-object-resource`? Client-side generation rules and parsing leniency requirements are deferred.
>
> Tests in this suite depend on having an RFC 4791 calendar collection to PUT into. They are expected to run after the rfc4791 suite bootstrap tests pass.
