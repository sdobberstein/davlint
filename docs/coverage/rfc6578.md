# RFC 6578 — WebDAV Sync-Collection

**Suite:** `rfc6578` | **Test file:** `internal/suites/rfc6578/rfc6578.go`

Defines the `DAV:sync-collection` REPORT, sync tokens as URIs, initial and subsequent sync semantics (added/changed/removed members), result truncation, and the `DAV:sync-token` live property. 23 tests covering all testable requirements.

## Deferred

| Requirement | Reason |
|-------------|--------|
| Client must treat sync-token as opaque (R-02) | Client-side rule |
| Malformed request body → 4xx (R-05/R-06) | Server error responses vary too widely to assert deterministically |
| sync-level:infinite + unsyncable child → 403 (R-15/R-16, R-23–R-25b) | Requires nested addressbook collections |
| Server MUST limit token invalidation window (R-18) | Not triggerable deterministically |

| Client assumes cascade on collection removal (R-37) | Client-side rule |
| Client must handle 507 (R-40) | Client-side; server behavior covered by R-38 |
| ACL-triggered change/removal (M01/M02) | Requires ACL manipulation; non-deterministic |
| Server MAY truncate without client request (M03) | Server-internal policy |

## Gaps

None — all testable requirements are covered.

## Depth notes
