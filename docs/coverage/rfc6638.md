# RFC 6638 — CalDAV Scheduling Extensions

**Suite:** not yet created

Defines CalDAV scheduling: scheduling inbox/outbox collections, organizer and attendee PUT processing with iTIP message delivery, SCHEDULE-TAG, free-busy POST to outbox, and scheduling privileges. Requires RFC 4791 (`calendar-access`) as a prerequisite.

## Deferred

All requirements are tagged `scheduling` — excluded from default protocol bundle runs. Tests execute when `--tag scheduling` is specified or when the server advertises `calendar-auto-schedule` in its DAV OPTIONS response.

| Reason | Applies to |
|--------|-----------|
| Tagged `scheduling` | All §2–§9 requirements: capability advertisement, inbox/outbox structure, organizer PUT processing (SCHEDULE-STATUS, PARTSTAT reset), attendee modification rules, SCHEDULE-TAG, incoming message processing, free-busy POST, scheduling privileges |

## Gaps

N/A — all work is blocked on the `scheduling` tag milestone. Prerequisite: rfc4791 suite must be passing. When the suite is created, proposed entry-point tests: `calendar-auto-schedule` in OPTIONS, inbox/outbox resourcetype, PUT with ATTENDEE sets SCHEDULE-STATUS, GET returns Schedule-Tag header, If-Schedule-Tag-Match mismatch → 412.
