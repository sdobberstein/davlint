# RFC 7986 — New Properties for iCalendar

**Suite:** `rfc7986` — 5 tests

Defines new iCalendar properties: `NAME`, `DESCRIPTION`, `COLOR`, `URL`,
`REFRESH-INTERVAL`, `SOURCE` (for VCALENDAR), and `CONFERENCE`, `IMAGE`
(for calendar components). Tests follow a PUT/GET round-trip pattern:
a CalDAV server must preserve stored calendar object data unchanged
(RFC 4791 §5.3.2), so any accepted object must be returned verbatim.

## Tests

| Test ID | Severity | Description |
|---------|----------|-------------|
| `rfc7986.color-vevent-roundtrip` | Must | PUT VEVENT with COLOR; GET returns COLOR |
| `rfc7986.name-vcalendar-roundtrip` | Must | PUT VCALENDAR with NAME; GET returns NAME |
| `rfc7986.description-vcalendar-roundtrip` | Must | PUT VCALENDAR with DESCRIPTION; GET returns DESCRIPTION |
| `rfc7986.url-vcalendar-roundtrip` | Must | PUT VCALENDAR with URL; GET returns URL |
| `rfc7986.conference-vevent-roundtrip` | Must | PUT VEVENT with CONFERENCE; GET returns CONFERENCE |

## Deferred

| Requirement | Reason |
|-------------|--------|
| IMAGE (§6.2) | Requires inline binary encoding (ENCODING=BASE64); not black-box testable via round-trip string match |
| REFRESH-INTERVAL (§5.2) | Server-side polling behavior; not observable via HTTP |
| SOURCE (§5.3) | Syndication hint; preservation tested implicitly by general round-trip guarantee |
| COLOR at VCALENDAR level (§5.4) | Component-level COLOR already covers the round-trip preservation property; VCALENDAR-level is additive |
