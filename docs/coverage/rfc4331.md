# RFC 4331 — Quota and Size Properties for Distributed Authoring and Versioning (DAV)

**Suite:** not yet created

Defines two live DAV properties: `DAV:quota-available-bytes` (free space remaining for the principal) and `DAV:quota-used-bytes` (space currently consumed). Servers that implement quota management SHOULD expose these.

## Standalone suite verdict

**Marginal — create only if target servers are known to implement it.** The properties are SHOULD-level and optional; many lightweight CardDAV/CalDAV servers (Radicale, Baikal) don't implement them. Tests would be:

1. PROPFIND on addressbook collection for both properties → if 404 in propstat, skip (server doesn't support)
2. If present, values are non-negative integers
3. PUT a resource → `quota-used-bytes` either increases or stays the same (some servers batch updates)

The skip-if-absent pattern means the suite is a no-op on most real-world targets. Worth adding once there's a concrete target server that implements quota.

## Deferred

| Requirement | Reason |
|-------------|--------|
| `quota-available-bytes` accuracy | Server-internal; can only check non-negative integer, not correctness |
| `quota-used-bytes` increases after PUT | Server may batch quota updates; not deterministic |
