# RFC 7809 — CalDAV: Time Zones by Reference

**Suite:** not yet created

Allows CalDAV servers to omit `VTIMEZONE` components when the time zone is a well-known standard zone from an associated RFC 7808 service. Advertised via `calendar-no-timezone` in the DAV header. **Skip entire suite if server does not advertise `calendar-no-timezone` in OPTIONS.**

## Deferred

| Requirement | Reason |
|-------------|--------|
| Server must have associated RFC 7808 timezone services (R-02) | Implementation-internal; not observable via HTTP |
| All §4 requirements | Client-side rules |

## Gaps

No tests yet. Priority areas when the suite is created (all conditional on `calendar-no-timezone` in OPTIONS):

| Area | Priority |
|------|----------|
| OPTIONS advertises `calendar-no-timezone` | First — gate for all other tests |
| `CalDAV-Timezones: T` → VTIMEZONE included in GET response | High |
| `CalDAV-Timezones: F` → standard zone VTIMEZONE omitted; non-standard zone VTIMEZONE preserved | High |
| Unknown TZID without embedded VTIMEZONE → rejected with `CALDAV:valid-timezone` | High |
| `CALDAV:calendar-timezone-id` present on calendar collections | Medium |
| PROPPATCH sets `calendar-timezone-id` → `calendar-timezone` synced (and vice versa) | Medium |
| PROPPATCH with unknown timezone ID → `CALDAV:valid-timezone` | Medium |
| calendar-query with `CALDAV:timezone-id` instead of full VTIMEZONE | Low |
