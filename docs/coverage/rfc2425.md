# RFC 2425 — MIME Content-Type for Directory Information

**Suite:** covered within `rfc2426` | no standalone suite planned

Defines the MIME directory format that vCard 3.0 (RFC 2426) is built on, including the line-folding rule (CRLF + SPACE or CRLF + TAB as a fold indicator, §5.8.1). RFC 2426 explicitly delegates folding handling to RFC 2425.

## Coverage

The single testable server-side requirement is covered within the `rfc2426` suite:

| Test | Requirement |
|------|-------------|
| `rfc2426.folded-line-parsed` | CRLF+SPACE fold indicator unfolded before parsing (§5.8.1 MUST) |
| `rfc2426.folded-line-tab-parsed` | CRLF+TAB fold indicator unfolded before parsing (§5.8.1 MUST) |

Both tests carry `RFC 2425 §5.8.1` as a co-reference.

## Standalone suite verdict

**Not warranted.** RFC 2425 is the foundation format spec; its only testable server-side rule (line folding) is already exercised as part of vCard 3.0 parsing. All other RFC 2425 requirements are either client-side authoring rules or content-type framing that is not observable via HTTP responses.
