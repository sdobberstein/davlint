# RFC 6350 — vCard 4.0

**Suite:** `rfc6350` | **Test file:** `internal/suites/rfc6350/rfc6350.go`

Defines the vCard 4.0 format. Tests validate server behavior observable via HTTP: rejecting malformed vCards on PUT and verifying correct content on GET. Suite is largely unimplemented — only 2 thin tests exist.

## Deferred

| Requirement | Reason |
|-------------|--------|
| Multi-octet chars must not split across fold boundaries (R-05) | Implementation-internal; not observable in HTTP responses |
| Dates must use ISO 8601 basic format (§4.3) | Format parsing rule; server enforcement not specified |
| INTEGER bounded to signed 64-bit; FLOAT to IEEE binary64 (§4.5/§4.6) | Implementation-internal numeric constraints |
| LANGUAGE parameter semantics (§5.1) | Not a server conformance test |
| ALTID must not appear on single-instance properties (§5.4) | Client-side authoring rule |
| CLIENTPIDMAP / PID source identifiers (§5.5/§6.7.7) | Synchronisation/merge protocol; client-side |
| vCard instance and property instance matching (§7.1) | Synchronisation/merge protocol |
| XML property preservation (§6.1.5) | Not observable via simple GET round-trip |

## Gaps

Most of this suite is unimplemented. Priority areas:

| Area | What's missing |
|------|----------------|
| Charset | `charset=latin1` rejected; `charset=utf-8` accepted |
| Line folding | CRLF+SP and CRLF+HT folds unfolded before parsing (parallel to rfc2426 tests) |
| Required properties | Missing FN → rejected; missing VERSION → rejected |
| VERSION placement | VERSION must immediately follow BEGIN:VCARD |
| VERSION value | VERSION:3.0 or VERSION:2.1 → rejected at 4.0 endpoint |
| BEGIN/END | Missing BEGIN or END → rejected |
| Value escaping | `,`, `;`, `\\`, `\n` round-trips |
| ADR structure | Sparse ADR preserves separator count |
| UID uniqueness | Second PUT with same UID on different URL → conflict |
| PREF range | PREF=0 or PREF=101 → rejected |
| KIND + MEMBER | MEMBER without `KIND:group` → rejected |

## Depth notes

- Only 2 tests exist: `uid-assignment` (server assigns UID when absent) and `get-default-v4` (GET returns VERSION:4.0) — both are lint-only
