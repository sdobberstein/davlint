# RFC 5545 — iCalendar Format

**Suite:** `rfc5545` — 13 tests

Defines the iCalendar data format. The server-testable surface is whether the server accepts valid iCalendar and rejects invalid iCalendar on PUT (returning `CALDAV:valid-calendar-data`). Tests depend on having an RFC 4791 calendar collection to PUT into.

## Tests

| Test ID | Severity | Description |
|---------|----------|-------------|
| `rfc5545.valid-vevent-accepted` | Must | PUT valid VCALENDAR+VEVENT is accepted |
| `rfc5545.vtodo-valid-accepted` | Must | PUT valid VCALENDAR+VTODO is accepted |
| `rfc5545.vcalendar-missing-version` | Must | VCALENDAR without VERSION → 4xx |
| `rfc5545.vcalendar-missing-prodid` | Must | VCALENDAR without PRODID → 4xx |
| `rfc5545.vevent-missing-uid` | Must | VEVENT without UID → 4xx |
| `rfc5545.vevent-missing-dtstamp` | Must | VEVENT without DTSTAMP → 4xx |
| `rfc5545.vevent-dtend-duration-exclusive` | Must | VEVENT with DTEND + DURATION → 4xx |
| `rfc5545.vevent-dtstart-dtend-type-mismatch` | Should | DTSTART DATE + DTEND DATE-TIME → 4xx |
| `rfc5545.vtimezone-referenced-but-missing` | Should | TZID referenced without VTIMEZONE → 4xx |
| `rfc5545.vtodo-due-duration-exclusive` | Must | VTODO with DUE + DURATION → 4xx |
| `rfc5545.rrule-until-count-exclusive` | Should | RRULE with UNTIL + COUNT → 4xx |
| `rfc5545.valarm-missing-action` | Must | VALARM without ACTION → 4xx |
| `rfc5545.valarm-missing-trigger` | Must | VALARM without TRIGGER → 4xx |
