# RFC 6352 — CardDAV

**Suite:** `rfc6352` | **Test file:** `internal/suites/rfc6352/rfc6352.go`

Defines CardDAV: address book collections, address object resources (vCard PUT/GET with preconditions), the addressbook-query and addressbook-multiget REPORTs, collations, and unauthenticated access enforcement.

## Deferred

| Requirement | Reason |
|-------------|--------|
| WebDAV ACL support (R-03) | Tagged `acl` — excluded by default |
| TLS support (R-04) | TLS-level; out of scope for HTTP black-box testing |
| Server MAY restrict addressbook creation locations (R-33) | Internal server policy; not observable |
| Server MAY reject changes to its own X- properties (R-47) | Permissive; server-specific |
| MUST NOT return strong ETag when data normalized (R-52) | Only observable if server normalizes on PUT; hard to trigger deterministically |
| Server MAY advertise reports on ordinary collections (R-62) | Permissive |
| Server MAY truncate results (R-90) | Permissive |
| CR omission / partial vCard validity in responses (R-105/106) | Formatting/rendering detail |
| Result truncation structure: R-91–R-93 (507 sub-status, DAV:error body, partial results) | Not triggerable deterministically from client side |

## Gaps

| Area | What's missing |
|------|----------------|
| DAV:expand-property REPORT (R-63) | Not tested at all |
| Non-standard names in address-data (R-77) | REPORT with unknown vCard prop name should be accepted |
| addressbook-query: Depth header behavior (R-82), unsupported media type in request (R-86) | Not tested |
| addressbook-multiget: Depth:nonzero → 400 (R-97), unsupported media type (R-101) | Not tested |
| GET with unsupported Accept type | Should return `CARDDAV:supported-address-data-conversion` |
| COPY/MOVE preconditions | R-39–R-41, R-45 tested for PUT only; COPY and MOVE paths not covered |

## Depth notes

- All PUT preconditions (supported-address-data, valid-address-data, no-uid-conflict, max-resource-size): one test each; combinations not tested; COPY/MOVE paths not covered
- addressbook-query and addressbook-multiget: response structure spot-checked; not exhaustively verified across filter variants
- Collation tests exercise the query round-trip but do not verify output ordering
- UID uniqueness (R-13) and addressbook-collection-location-ok (R-44) are lint-only; second-write conflicts not verified
