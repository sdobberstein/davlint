# RFC 6351 — xCard: An XML Representation of vCard

**Suite:** not yet created

Defines an XML representation of vCard using the media type `application/vcard+xml`. A CardDAV server MAY support this alongside `text/vcard` via HTTP content negotiation.

## Standalone suite verdict

**Low priority — same reasoning as rfc7095 (jCard).** xCard support is a MAY; virtually no production CardDAV servers implement it. The test surface is identical in structure to jCard: GET with `Accept: application/vcard+xml`, validate XML structure, PUT with `Content-Type: application/vcard+xml`. Skip entirely unless a target server advertising xCard is identified.
