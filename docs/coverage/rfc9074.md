# RFC 9074 — VALARM Extensions for iCalendar

**Suite:** not yet created — blocked on rfc4791

Extends VALARM with `ACKNOWLEDGED` (tracks when an alarm was last dismissed) and `PROXIMITY` (location-based triggers) properties. `ACKNOWLEDGED` in particular is widely used by modern calendar clients to sync alarm dismissal state across devices.

## Standalone suite verdict

**Moderate priority — after rfc4791.** `ACKNOWLEDGED` is testable: PUT a VEVENT with a VALARM containing `ACKNOWLEDGED` → GET and verify the property survives the round-trip. `PROXIMITY` is harder (location-trigger semantics are server-internal). A small suite of 2–3 round-trip tests is feasible once the rfc4791 CalDAV infrastructure exists. Worth doing for completeness since `ACKNOWLEDGED` is common in real-world calendar data.
