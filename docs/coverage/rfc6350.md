# RFC 6350 — vCard 4.0

**Suite:** `rfc6350` | **Test file:** `internal/suites/rfc6350/rfc6350.go`

Defines the vCard 4.0 format. Tests validate server behavior observable via HTTP: rejecting malformed vCards on PUT and verifying correct content on GET. 27 tests covering structural validation, charset enforcement, folding, escaping, and 4.0-specific properties.

## Deferred

| Requirement | Reason |
|-------------|--------|
| Multi-octet chars must not split across fold boundaries (R-05) | Implementation-internal; not observable in HTTP responses |
| Dates must use ISO 8601 basic format (§4.3) | Format parsing rule; server enforcement not specified |
| INTEGER bounded to signed 64-bit; FLOAT to IEEE binary64 (§4.5/§4.6) | Implementation-internal numeric constraints |
| LANGUAGE parameter semantics (§5.1) | Not a server conformance test |
| ALTID must not appear on single-instance properties (§5.4) | Client-side authoring rule |
| CLIENTPIDMAP / PID source identifiers (§5.5/§6.7.7) | Synchronisation/merge protocol; client-side |
| vCard instance and property instance matching (§7.1) | Synchronisation/merge protocol |
| XML property preservation (§6.1.5) | Not observable via simple GET round-trip |

## Gaps

| Area | What's missing |
|------|----------------|
| Server output folding | Lines >75 chars SHOULD be folded in GET response — not a server MUST; no observable failure condition |

## Depth notes

- Charset: `charset=latin1` rejection tested; `charset=utf-8` acceptance confirmed
- Folding: CRLF+SP and CRLF+HT both covered via FN round-trip
- VERSION: missing, wrong-order, 3.0, and 2.1 all tested
- Escaping: `,`, `;`, `\\`, `\n` all covered; backslash test asserts "path" survives (either encoded or decoded form is valid)
- PREF: out-of-range (0 and 101) tested; valid range (1–100) implicitly accepted via other tests
- `member-without-kind-group` is registered as `Should` — RFC 6350 §6.6.5 says "is to be used" (no explicit MUST reject)
- `kind-unknown-rejected` is registered as `Must` — RFC 6350 §6.7.9 defines an exhaustive list of valid values
