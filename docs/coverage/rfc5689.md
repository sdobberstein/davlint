> **Context:** RFC 5689 extends plain MKCOL (RFC 4918 ┬з9.3) to accept a request body that specifies the new collection's resource type and initial properties. This is the standard way to create typed collections тАФ CalDAV calendars and CardDAV address books тАФ without requiring protocol-specific methods (MKCALENDAR). RFC 4791 ┬з5.3.1 and RFC 6352 ┬з6.3.1 both reference extended MKCOL as the preferred creation mechanism.

## ┬з4 тАФ Integration with CalDAV and CardDAV

RFC 5689 ┬з4 states that when a server supports both extended MKCOL and a protocol-specific creation method (e.g. MKCALENDAR for CalDAV), it MUST support using extended MKCOL as an equivalent alternative.
