# RFC 5397 — WebDAV Current Principal Extension

**Suite:** covered within `rfc6764` | no standalone suite planned

Defines the `DAV:current-user-principal` property, which returns the principal URL of the authenticated user making the request. This is the first step in the RFC 6764 discovery chain (context path → current-user-principal → home set).

## Coverage

All testable requirements are covered within the `rfc6764` suite, which uses `DAV:current-user-principal` as part of the full discovery chain:

| Test | Requirement |
|------|-------------|
| `rfc6764.current-user-principal` | Authenticated PROPFIND returns the property (§3 MUST) |
| `rfc6764.current-user-principal-requires-auth` | Unauthenticated request returns 401 (§3 / RFC 6764 §7 MUST) |

Both tests carry `RFC 5397 §3` as a co-reference.

## Standalone suite verdict

**Not warranted.** The RFC is a single short property definition. Its two testable requirements are already covered inline within `rfc6764`, which is the natural home for discovery chain tests. Splitting it out would add a suite with 2 tests that duplicate existing ones.
