## §8 (DAV Header — RFC 4918 cross-reference)

The existing `rfc6764.dav-header` test (OPTIONS returns `DAV: 1, 2, access-control`) is correctly placed here as part of the discovery chain verification but the requirement originates in RFC 4918 §10.1. It will also be covered in `rfc4918` conformance tests.

## TLS Requirements (R-07-5, R-07-6, R-08-7, R-08-8, R-08-9)

Several RFC 6764 §7–8 requirements involve TLS certificate verification — SRV-ID and DNS-ID identifiers, SSL 2.0 rejection, RFC 6125 certificate chain validation.

These are **deferred** from HTTP black-box testing. They require TLS-level inspection (parsing the server's presented certificate, attempting deprecated protocol negotiation), which is a distinct testing concern from the HTTP conformance tests this tool focuses on.

| Req | Requirement | Reason deferred |
|-----|-------------|-----------------|
| R-07-5 | TLS certificate MUST be verifiable per RFC 6125 | Requires TLS-level certificate inspection |
| R-07-6 | Certificate SHOULD include SRV-ID and DNS-ID | Requires TLS-level certificate inspection |
| R-08-7 | Certificate MUST include SRV-ID when SRV FQDN ≠ queried domain | Requires TLS-level inspection |
| R-08-8 | Certificate MUST include SRV-ID or DNS-ID | Requires TLS-level inspection |
| R-08-9 | Server MUST NOT accept SSL 2.0 | Requires TLS-level protocol negotiation test |

## Deferred (DNS-level / Client-side)

The majority of RFC 6764's normative requirements describe **client bootstrapping procedures** and **DNS record configuration**. These are not testable by directing HTTP requests at a server.

| Req | Section | Requirement | Reason |
|-----|---------|-------------|--------|
| R-03-1 | §3 | Clients MUST honor SRV Priority/Weight | Client-side DNS behavior |
| R-04-1 | §4 | TXT record `path` key MUST be the context path | DNS record content |
| R-04-2 | §4 | Clients MUST use TXT `path` key as context path | Client-side rule |
| R-06-1–6 | §6 | All SRV lookup, TXT record, and TLS-first fallback client behaviors | DNS-level / client-side |
| R-06-8 | §6 | Clients MUST handle redirects on PROPFIND to context path | Client-side rule |
| R-06-11–12 | §6 | Client caching and SRV retry on failure | Client-side rule |
| R-07-2–3 | §7 | Auth identifier uniqueness/conflict rules | Server configuration policy; not observable via HTTP |
| R-08-1–6 | §8 | Client TLS preference and certificate verification behaviors | Client-side / DNS-level |
