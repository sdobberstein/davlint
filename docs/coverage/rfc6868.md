# RFC 6868 — Parameter Value Encoding in iCalendar and vCard

**Suite:** `rfc6868` — 6 tests

Defines caret (`^`) escape sequences for property parameter values in both
iCalendar (RFC 5545) and vCard (RFC 2426 / RFC 6350):

- `^^` → literal caret
- `^n` → newline (U+000A)
- `^'` → double-quote (U+0022)
- Unknown sequences (e.g. `^x`) MUST be passed through unchanged (§4).

Tests follow a PUT→GET round-trip pattern, verifying that caret sequences in
parameter values do not corrupt the property value. Both the vCard addressbook
and iCalendar calendar surfaces are exercised.

## Tests

| Test ID | Severity | Description |
|---------|----------|-------------|
| `rfc6868.vcard-caret-caret-roundtrip` | Must | `^^` in vCard parameter; FN value intact |
| `rfc6868.vcard-caret-n-roundtrip` | Must | `^n` in vCard parameter; FN value intact |
| `rfc6868.vcard-caret-quote-roundtrip` | Must | `^'` in vCard parameter; FN value intact |
| `rfc6868.vcard-caret-unknown-accepted` | Must | `^x` in vCard parameter; server accepts (not 4xx) |
| `rfc6868.icalendar-caret-caret-roundtrip` | Must | `^^` in iCalendar parameter; SUMMARY value intact |
| `rfc6868.icalendar-caret-quote-roundtrip` | Must | `^'` in iCalendar parameter; SUMMARY value intact |

## Deferred

| Requirement | Reason |
|-------------|--------|
| Server re-encodes output in caret form | Output format is implementation-defined; server may decode to native and not re-encode |
| `^n` in iCalendar parameter | iCalendar multiline parameter values are rare; `^^` and `^'` cover the common cases |
