# RFC 3744 — WebDAV Access Control (ACL)

**Suite:** not yet created

Defines the WebDAV ACL protocol: privilege model, principal resources, access control properties, the ACL method, and ACL REPORT methods. Required by both RFC 4791 (CalDAV) and RFC 6352 (CardDAV).

## Deferred

All requirements are tagged `acl` — excluded from default protocol bundle runs. Tests execute when `--tag acl` is specified or when the server advertises `access-control` in its DAV OPTIONS response.

| Reason | Applies to |
|--------|-----------|
| Tagged `acl` | All §3–§9 requirements: privilege model, principal resources, ACL properties, ACL method, ACL precondition codes, COPY/MOVE interaction, ACL REPORTs |

## Gaps

N/A — all work is blocked on the `acl` tag milestone. When the suite is created, proposed test areas include: `access-control` in OPTIONS DAV header, principal resourcetype, ACL properties presence and allprop exclusion, ACL method modifies permissions, precondition codes (no-abstract, recognized-principal, no-protected-ace-conflict), and the ACL REPORT methods (acl-principal-prop-set, principal-match, principal-property-search, principal-search-property-set).
