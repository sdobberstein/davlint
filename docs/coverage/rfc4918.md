# RFC 4918 — WebDAV Core

**Suite:** `rfc4918` | **Test file:** `internal/suites/rfc4918/rfc4918.go`

Defines the core WebDAV methods (OPTIONS, MKCOL, PUT, GET, DELETE, PROPFIND, PROPPATCH, COPY, MOVE), live properties, ETags, and the Depth/Overwrite/If headers used by both CalDAV and CardDAV.

## Deferred

| Requirement | Reason |
|-------------|--------|
| XML namespace compliance (§8.2-3) | Implementation detail; not externally observable |
| Client behavior on bad XML response (§8.2-6) | Client-side rule |
| Infinity PROPFIND MAY be disabled (§9.1-4) | Permissive; CardDAV requires rejection — tested as conformance |
| POST on collection (§9.5-1) | Server-defined behavior; no normative test surface |
| Partial DELETE MAY return 207 (§9.6-3) | Permissive; not triggerable deterministically |
| Timeout header MAY be ignored (§10.7) | Permissive |
| 507 Insufficient Storage (§11.5) | Server resource condition; not triggerable |
| LOCK / UNLOCK (§9.10–9.11), Lock-Token header (§10.5), §7 locking semantics | Tagged `locking` — excluded by default; runs when `--tag locking` or DAV header includes `2` |

## Gaps

| Area | What's missing |
|------|----------------|
| PROPPATCH | Protected property → 403 in propstat; atomicity (one valid + one protected → neither changes) |
| COPY | Overwrite absent behaves as T; dead property preservation at destination |
| MOVE | Overwrite:F → 412; Overwrite absent behaves as T |
| MKCOL | Missing parent → 409 |
| PUT | To collection URL → 405; missing parent → 409 |
| Destination header | COPY/MOVE without Destination → 400 |
| If-Match on PUT | Matching ETag succeeds; mismatched → 412; If-None-Match:* → 412 when resource exists |
| §15 live properties | `resourcetype` has `<collection/>` on collections; `getetag` matches ETag header; `getlastmodified` is RFC 1123 format |

## Depth notes

- PROPFIND variants (depth 0/1, propname, allprop): response status and property presence checked; values not cross-validated
- PROPPATCH set/remove: 207 status checked; property value correctness not verified on GET
- DELETE collection: 204 + collection 404 checked; member enumeration not verified
- Depth:infinity rejection is CardDAV-specific behavior (MUST); general RFC 4918 SHOULD is not covered
- If-Match tests cover DELETE only; PUT and other methods not tested here (see rfc7232 suite)
