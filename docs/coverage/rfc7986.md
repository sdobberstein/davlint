# RFC 7986 — New Properties for iCalendar

**Suite:** not yet created — blocked on rfc4791

Extends iCalendar (RFC 5545) with new calendar/component-level properties: `COLOR`, `CONFERENCE`, `IMAGE`, `REFRESH-INTERVAL`, `SOURCE`, `URL`, and `NAME`/`DESCRIPTION` at the VCALENDAR level. Used by CalDAV servers that support rich calendar metadata.

## Standalone suite verdict

**Yes — worth a suite, after rfc4791 exists.** These properties are increasingly common in modern CalDAV servers (FastMail, Google, iCloud advertise COLOR and NAME). The test surface is straightforward: PUT a VEVENT/VCALENDAR containing these properties → GET and verify they survive the round-trip. Blocked entirely until the rfc4791 suite provides the CalDAV infrastructure.

## Testable requirements (sketch)

| Property | Test sketch |
|----------|-------------|
| `COLOR` | PUT VCALENDAR with `COLOR:red` → GET contains COLOR |
| `NAME` | PUT VCALENDAR with `NAME:My Calendar` → GET contains NAME |
| `DESCRIPTION` (VCALENDAR-level) | PUT with calendar-level DESCRIPTION → survives round-trip |
| `CONFERENCE` | PUT VEVENT with CONFERENCE URI → survives round-trip |
| `URL` (VCALENDAR-level) | PUT with calendar-level URL → survives round-trip |

## Deferred

| Requirement | Reason |
|-------------|--------|
| `IMAGE` with inline binary | Requires ENCODING=BASE64; same issue as rfc2426 binary props |
| `REFRESH-INTERVAL` semantics | Server-side scheduling behavior; not observable via HTTP |
