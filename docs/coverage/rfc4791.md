# RFC 4791 — CalDAV Core

**Suite:** `rfc4791` — 21 tests

Defines CalDAV: calendar collections, calendar object resources (iCalendar), MKCALENDAR, the calendar-query, calendar-multiget, and free-busy-query REPORTs, PUT preconditions, collations, and ETags on calendar resources.

## Deferred

| Requirement | Reason |
|-------------|--------|
| WebDAV ACL (R-03, R-41–R-44) | Tagged `acl` — excluded by default |
| TLS (R-04) | TLS-level; out of scope |
| Clients SHOULD use `If-None-Match: *` when creating (S-01) | Client-side recommendation |
| MAY support `limit-recurrence-set`, `expand`, `limit-freebusy-set` in calendar-data | MAY-level partial retrieval options |
| MAY support X- component/property names in filters | Server MAY reject — failure mode is correct server behavior |
| `CALDAV:calendar-description` property | MAY be protected; property existence optional |
| Collations: ascii-casemap and octet, unsupported → CALDAV:supported-collation | Not covered — medium priority future addition |
