# RFC 7953 — Calendar Availability (VAVAILABILITY)

**Suite:** not yet created

Defines the `VAVAILABILITY` iCalendar component for expressing user availability. Can be stored as a calendar object resource in a CalDAV collection, and optionally as the `CALDAV:calendar-availability` property on a scheduling inbox (when `calendar-auto-schedule` is advertised).

## Deferred

| Requirement | Reason |
|-------------|--------|
| `CALDAV:calendar-availability` property on scheduling Inbox (R-14) | Tagged `scheduling` — conditional on `calendar-auto-schedule` |
| iTIP free-busy aggregation of VAVAILABILITY (R-15) | Tagged `scheduling` |
| `calendar-availability` value structure: exactly one VAVAILABILITY, no other components (R-16/R-17) | Tagged `scheduling` — property only present when auto-schedule active |
| Free-busy output must not leak SUMMARY/LOCATION/DESCRIPTION (R-18) | Implementation-internal; not observable without scheduling infrastructure |
