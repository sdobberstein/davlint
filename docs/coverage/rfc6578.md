## §3.2 — sync-collection REPORT (Structure & Semantics)

| Req | Requirement | Current Test | Status |
|-----|-------------|--------------|--------|
| R01 | `DAV:sync-collection` MUST be listed in `DAV:supported-report-set` | `rfc6578.supported-report-set` | conformance ✓ |
| R03 | sync-token MUST be a valid URI with a scheme | `rfc6578.sync-token-is-uri` | conformance ✓ |
| R04 | Request-URI MUST identify a collection | `rfc6578.report-on-non-collection` | conformance ✓ |
| R07 | Depth header other than `0` MUST return 400 | `rfc6578.depth-nonzero-rejected` | conformance ✓ |
| R08 | Successful response body MUST be `DAV:multistatus` | `rfc6578.initial-sync` (indirectly) | lint only |
| R09 | `DAV:multistatus` MUST contain one `DAV:sync-token` element | `rfc6578.initial-sync` | conformance ✓ |
| R10 | A given member URL MUST appear only once in the response | `rfc6578.no-duplicate-hrefs` | conformance ✓ |
| R11 | Changed member response MUST have at least one `DAV:propstat` | `rfc6578.subsequent-sync-changed` | lint only |
| R12 | Changed member response MUST NOT have a `DAV:status` element | `rfc6578.subsequent-sync-changed` | lint only |
| R13 | Removed member response MUST have `DAV:status` 404 Not Found | `rfc6578.subsequent-sync-removed` | conformance ✓ |
| R14 | Removed member response MUST NOT have any `DAV:propstat` | `rfc6578.subsequent-sync-removed` | conformance ✓ |
| R17 | Invalid/unrecognized sync-token MUST trigger `DAV:valid-sync-token` precondition | `rfc6578.invalid-token-rejected` | lint only |

**Conformance gaps:**

`rfc6578.invalid-token-rejected` only verifies that a 4xx is returned. It does not verify that the response body contains a `DAV:error` element with `DAV:valid-sync-token`. That is the normative requirement.

`rfc6578.subsequent-sync-changed` verifies changed response structure only for a newly added resource. Does not verify for an updated resource (ETag change).

**Missing tests:**
- `rfc6578.invalid-token-error-element` — invalid token response body contains `DAV:error` with `DAV:valid-sync-token` child
- `rfc6578.subsequent-sync-updated` — PUT updating an existing resource; subsequent sync reports it as changed (R31)
- `rfc6578.multistatus-response-content-type` — 207 response Content-Type is `application/xml` or `text/xml`
