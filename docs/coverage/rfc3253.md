# RFC 3253 — DeltaV (Versioning Extensions to WebDAV)

**Suite:** not yet created

Defines two things: (1) the **REPORT method** (§3.6) — a generic WebDAV method used by CalDAV, CardDAV, ACL, and sync; and (2) the **DeltaV versioning feature set** — practically never implemented by modern CalDAV/CardDAV servers.

## Deferred

| Requirement | Reason |
|-------------|--------|
| All versioning requirements (§4+): VERSION-CONTROL, CHECKOUT, CHECKIN, version history, workspaces, auto-version, fork restrictions | Tagged `versioning` — DeltaV is not implemented by essentially any modern CalDAV/CardDAV server; excluded from all bundle runs |

## Gaps

The cross-cutting REPORT method requirements (§3.6) are not yet tested in a dedicated suite. Note that `DAV:supported-report-set` is already tested in the rfc4791 and rfc6352 suites — the rfc3253 suite should focus on the generic method contract.

| Area | What's missing |
|------|----------------|
| Unsupported report type → 403 with `DAV:supported-report` precondition body | Not tested anywhere |
| REPORT is idempotent and non-modifying | Two identical REPORTs return same result |
