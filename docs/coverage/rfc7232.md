> **Context:** RFC 7232 defines HTTP conditional request headers (`If-Match`, `If-None-Match`, `If-Modified-Since`, `If-Unmodified-Since`) and validator semantics (ETags, Last-Modified). In CalDAV/CardDAV these are critical for safe concurrent writes: clients MUST use `If-None-Match: *` when creating resources and `If-Match: <etag>` when updating, to prevent lost updates. RFC 6352 §5.3.4 and RFC 4791 §5.3.4 both require servers to enforce these semantics.
