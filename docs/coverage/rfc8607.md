# RFC 8607 — Calendaring Extensions to WebDAV (CalDAV): Managed Attachments

**Suite:** not yet created — blocked on rfc4791

Defines a mechanism for CalDAV servers to manage binary attachments as first-class DAV resources, referenced from calendar data via `ATTACH` properties with `MANAGED-ID` parameters. Uses POST on a calendar collection to create/update/remove attachments, and `Prefer: return=representation` to get the updated resource.

## Standalone suite verdict

**Deferred — create after rfc4791 and only if target servers advertise support.** Managed attachments require specific server support (advertised via `CALDAV:managed-attachments-server-info`). The protocol is complex (POST with `action=attachment-add`, `Prefer` header handling, `ATTACH` parameter round-trips). Worth a suite eventually for completeness, but low priority compared to core CalDAV.
